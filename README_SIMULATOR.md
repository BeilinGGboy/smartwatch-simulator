# 智能手表数据模拟器

一个高性能的 Go 语言数据模拟器，可以模拟 10 万级智能手表设备同时上传数据到服务器。

## 功能特性

- ✅ **大规模设备模拟**：支持 10 万+ 设备同时运行
- ✅ **智能数据生成**：生成符合真实生理规律的数据（心率、步数、睡眠、运动）
- ✅ **多种上传模式**：实时上传、批量上传、定时上传
- ✅ **高并发上传**：支持并发上传，可配置并发数
- ✅ **实时监控**：实时显示数据生成和上传统计
- ✅ **资源优化**：内存和 CPU 优化，支持长时间运行

## 项目结构

```
smartwatch-simulator/
├── simulator/
│   ├── config/          # 配置管理
│   ├── generator/        # 数据生成器
│   ├── device/          # 设备模拟器
│   └── uploader/        # 数据上传器
├── utils/               # 工具函数
├── main.go              # 主程序
├── test_server.go       # 测试服务器（用于接收数据）
```

## 快速开始

### 1. 安装依赖

```bash
go mod download
```

### 2. 启动测试服务器（可选）

在一个终端窗口运行：

```bash
go run test_server.go
```

服务器将在 `http://localhost:8080` 启动。

### 3. 运行模拟器

在另一个终端窗口运行：

```bash
go run main.go
```

## 配置说明

默认配置（可在 `simulator/config/config.go` 中修改）：

- **设备数量**: 100,000
- **数据生成间隔**: 1秒
- **批量上传大小**: 100条
- **上传间隔**: 5秒
- **服务器地址**: http://localhost:8080
- **并发上传数**: 100
- **上传模式分布**:
  - 实时上传: 20%
  - 批量上传: 70%
  - 定时上传: 10%

## 使用示例

### 基本使用

```bash
# 使用默认配置运行
go run main.go
```

### 修改配置

编辑 `simulator/config/config.go` 中的 `DefaultConfig()` 函数，或修改 `main.go` 中的配置加载逻辑。

模拟器运行时会每 5 秒自动打印统计信息，包括：
- 设备数量
- 数据生成量
- 数据上传量
- 成功率
- 数据量（MB）

如果启动了测试服务器，也可以访问：
```bash
curl http://localhost:8080/stats
```

## 数据生成策略

### 心率数据
- 正常模式: 60-100 BPM（在基础心率附近波动）
- 运动模式: 120-180 BPM
- 睡眠模式: 50-70 BPM

### 步数数据
- 遵循时间分布规律（早上少，下午多）
- 每日目标: 5000-15000 步
- 自动计算距离和卡路里

### 睡眠数据
- 睡眠时间: 晚上 22:00 - 23:30 开始
- 睡眠时长: 6-9 小时
- 包含深睡、浅睡比例

### 运动数据
- 支持跑步、骑行、步行、游泳
- 根据运动类型计算距离和卡路里
- 包含平均心率和最大心率

## 上传模式说明

### 实时上传模式（RealTime）
- 每次生成数据立即上传
- 适合：心率、GPS 轨迹等实时数据
- 特点：延迟低，但请求频率高

### 批量上传模式（Batch）
- 生成数据后缓存，定时批量上传
- 适合：步数、睡眠等非实时数据
- 特点：减少请求数，提高效率

### 定时上传模式（Timer）
- 定时上传所有缓存数据
- 适合：设备状态、汇总数据
- 特点：请求频率最低

## 性能指标

在默认配置下（10万设备）：
- **内存占用**: 约 500MB - 1GB
- **CPU 占用**: 中等（取决于并发数）
- **数据生成速度**: 约 10万条/秒
- **上传速度**: 取决于服务器性能和网络

## 停止模拟器

按 `Ctrl+C` 停止模拟器，程序会：
1. 停止所有设备
2. 等待数据上传完成
3. 显示最终统计信息

## 注意事项

1. **服务器地址**: 确保服务器地址正确，否则上传会失败
2. **并发数**: 根据服务器性能调整并发上传数
3. **内存**: 大规模运行时注意内存使用
4. **网络**: 确保网络连接稳定

## 扩展开发

### 添加新的数据类型

1. 在 `simulator/generator/data_generator.go` 中添加新的数据结构
2. 实现数据生成逻辑
3. 在 `GenerateDeviceData` 中添加新类型

### 自定义上传逻辑

1. 修改 `simulator/uploader/uploader.go`
2. 实现自定义的上传协议或格式

### 添加更多监控指标

1. 在 `device.Device` 和 `uploader.Uploader` 中添加统计字段
2. 在 `printStats` 函数中显示新指标

## 常见问题

**Q: 如何减少内存占用？**
A: 减少设备数量或增加批量上传比例。

**Q: 上传失败怎么办？**
A: 检查服务器是否运行，网络是否正常，服务器地址是否正确。

**Q: 如何测试服务器性能？**
A: 逐步增加设备数量和并发数，观察服务器响应时间和错误率。
