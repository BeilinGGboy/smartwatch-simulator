# 智能手表数据模拟器

一个高性能的 Go 语言数据模拟器，可以模拟 10 万级智能手表设备同时上传数据到服务器。

## 项目结构

```
smartwatch-simulator/
├── main.go                 # 主程序入口
├── go.mod                  # 依赖管理
├── go.sum                  # 依赖校验
│
├── simulator/              # 模拟器核心模块
│   ├── config/            # 配置管理
│   │   └── config.go
│   ├── generator/         # 数据生成器
│   │   └── data_generator.go
│   ├── device/           # 设备模拟器
│   │   └── device.go
│   └── uploader/         # 数据上传器
│       └── uploader.go
│
├── utils/                 # 工具函数
│   └── uuid.go
│
├── README.md             # 项目说明
└── ARCHITECTURE.md       # 架构文档
```

## 功能特性

- ✅ **大规模设备模拟**：支持 10 万+ 设备同时运行
- ✅ **智能数据生成**：生成符合真实生理规律的数据
- ✅ **多种上传模式**：实时上传、批量上传、定时上传
- ✅ **高并发上传**：支持并发上传，可配置并发数
- ✅ **实时监控**：实时显示数据生成和上传统计
- ✅ **资源优化**：内存和 CPU 优化，支持长时间运行

## 快速开始

### 1. 安装依赖

```bash
go mod download
```

### 2. 配置服务器地址

编辑 `simulator/config/config.go` 中的 `ServerURL`：

```go
ServerURL: "http://your-server-ip:8080",
```

### 3. 运行模拟器

```bash
go run main.go
```

## 配置说明

默认配置（可在 `simulator/config/config.go` 中修改）：

- **设备数量**: 100,000
- **数据生成间隔**: 1秒
- **批量上传大小**: 100条
- **上传间隔**: 5秒
- **服务器地址**: http://localhost:8080
- **并发上传数**: 100
- **上传模式分布**:
  - 实时上传: 20%
  - 批量上传: 70%
  - 定时上传: 10%

## 使用示例

### 基本使用

```bash
# 使用默认配置运行
go run main.go
```

### 修改配置

编辑 `simulator/config/config.go` 中的 `DefaultConfig()` 函数。

### 查看统计

模拟器运行时会每 5 秒自动打印统计信息。

## 数据生成策略

### 心率数据
- 正常模式: 60-100 BPM
- 运动模式: 120-180 BPM
- 睡眠模式: 50-70 BPM

### 步数数据
- 遵循时间分布规律（早上少，下午多）
- 每日目标: 5000-15000 步

### 睡眠数据
- 睡眠时间: 晚上 22:00 - 23:30 开始
- 睡眠时长: 6-9 小时

### 运动数据
- 支持跑步、骑行、步行、游泳
- 根据运动类型计算距离和卡路里

## 上传模式说明

### 实时上传模式（RealTime）
- 每次生成数据立即上传
- 适合：心率、GPS 轨迹等实时数据

### 批量上传模式（Batch）
- 生成数据后缓存，定时批量上传
- 适合：步数、睡眠等非实时数据

### 定时上传模式（Timer）
- 定时上传所有缓存数据
- 适合：设备状态、汇总数据

## 性能指标

在默认配置下（10万设备）：
- **内存占用**: 约 500MB - 1GB
- **CPU 占用**: 中等
- **数据生成速度**: 约 10万条/秒
- **上传速度**: 取决于服务器性能和网络

## 停止模拟器

按 `Ctrl+C` 停止模拟器，程序会：
1. 停止所有设备
2. 等待数据上传完成
3. 显示最终统计信息

## 注意事项

1. **服务器地址**: 确保服务器地址正确
2. **并发数**: 根据服务器性能调整并发上传数
3. **内存**: 大规模运行时注意内存使用
4. **网络**: 确保网络连接稳定
